# Code of Conduct

The ACT (Agent Content Tree) project adopts the **Contributor Covenant, version 2.1**, as its code of conduct. The full canonical text is available at:

<https://www.contributor-covenant.org/version/2/1/code_of_conduct/>

We adopt that document by reference rather than reproducing it here, so the version we follow always tracks the upstream wording.

## Scope

This Code of Conduct applies to all project spaces — the `act-spec/act` repository, its issue and pull request threads, the spec proposal (ASP) process, and any official communication channels run by the project. It also applies when an individual is representing the project in public, for example at a conference talk, in a meetup, on social media as a maintainer, or in any forum where they are identified as acting on behalf of ACT.

## Reporting

If you experience or witness behavior that violates this Code of Conduct, please report it privately by email to **maintainers@act-spec.org**. Reports are kept confidential and reviewed by the project maintainers. You do not need to be certain that something is a violation to report it; if it bothered you, that is reason enough to write.

When reporting, include as much detail as you are comfortable sharing: what happened, when and where it happened, who was involved, any links or screenshots, and whether you would like to remain anonymous to the person being reported.

## Enforcement

Maintainers will review reports promptly and confidentially. After reviewing a report, maintainers may take any action they consider appropriate, ranging from a private clarification, to a public correction, to a temporary suspension from project spaces, to a permanent ban for repeated or severe violations. Maintainers will explain their reasoning to the reporter and to the person being reported wherever doing so does not compromise anyone's safety or privacy.

Maintainers themselves are held to the same standard. A report involving a maintainer should still be sent to **maintainers@act-spec.org**; the maintainer in question will recuse from handling it.
